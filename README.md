# Implementation Plan - ClassTrack Student Attendance Management System

ClassTrack is a MERN stack web application for student attendance management. It features separate Admin and Student dashboards, interactive attendance marking, detailed reports with CSV/PDF exports, and robust role-based access control.

---

## Technical Stack & Libraries

### Backend
- **Node.js & Express.js**: Server environment and RESTful API framework.
- **MongoDB & Mongoose**: NoSQL database for data storage and schema validation.
- **jsonwebtoken (JWT)**: Secure user session tokens.
- **bcryptjs**: Password hashing.
- **cors & dotenv**: Cross-origin requests and environment variable configuration.

### Frontend
- **React.js (Vite)**: Modern fast frontend tooling.
- **Tailwind CSS**: Utility-first CSS framework for clean, modern academic theme styles.
- **React Router DOM**: Client-side routing for public and protected paths.
- **Axios**: Promised-based client for REST API communication.
- **Lucide React**: Premium iconography.
- **Recharts**: Responsive chart library for attendance trend visualization.
- **jspdf & jspdf-autotable**: Client-side high-quality PDF report generation.

## Open Questions

- **Student Password Self-Reset**: Should students have the ability to reset their password via email? For this project, we plan to implement a simple profile editing panel where the student (or admin) can update their password once logged in. Let us know if you require SMTP email integration.
- **Late Attendance Weight**: Does a status of "Late" count towards attendance percentage? We plan to count "Present" as 100%, "Late" as 100% (or 50%, adjustable), and "Absent" as 0% for percentage calculation. We will default to treating "Late" as present for percentage calculations, but display it separately.

## Database Schemas

### Admin Schema (`backend/models/Admin.js`)
- `name`: `String` (required)
- `email`: `String` (required, unique, lowecase)
- `password`: `String` (required, hashed)
- `role`: `String` (default: `'admin'`)

### Student Schema (`backend/models/Student.js`)
- `name`: `String` (required)
- `rollNumber`: `String` (required, unique, uppercase)
- `department`: `String` (required)
- `year`: `String` (required)
- `section`: `String` (required)
- `email`: `String` (required, unique, lowercase)
- `phone`: `String` (required)
- `password`: `String` (required, hashed)
- `role`: `String` (default: `'student'`)

### Attendance Schema (`backend/models/Attendance.js`)
- `studentId`: `ObjectId` (ref `'Student'`, required)
- `date`: `Date` (required, normalized to midnight UTC to prevent duplicates)
- `status`: `String` (enum: `['Present', 'Absent', 'Late']`, required)
- `markedBy`: `ObjectId` (ref `'Admin'`, required)
